  const apiKey = process.env.OPENAI_API_KEY
  if (!apiKey || apiKey.trim() === '') {
    return fallbackTitle // First user message
  }

  const completion = await openai.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [/* system + user prompts */],

    max_tokens: 20,
  })

  return completion.choices[0].message.content?.trim() || ''

## 🔧 Technical Details

### Title Caching Strategy
**Format:** JSON file per archive directory
**Location:** `.titles.json` in archive folders
**Structure:**
```json
{
  "archive_filename_1.json": "Generated Title Here",
  "archive_filename_2.json": "Another Title"
}
```

**Cache Update Logic:**
1. Check if title exists in cache
2. If not, generate using GPT-4o-mini
3. Store in cache object
4. Write cache to disk
5. Return cached titles on subsequent loads

### Archive Deletion Flow
1. User clicks delete button → confirmation modal
2. User confirms → `DELETE /api/{type}/{name}/archives/{filename}`
3. Server deletes archive file
4. Server removes entry from `.titles.json`
5. Client updates UI state
6. If viewing deleted archive, close viewer

### Performance Optimizations
- Parallel title generation with `Promise.all()`
- Single cache write per archive fetch
- Lazy loading of archive content (only when viewed)
- 5-minute cache TTL consideration (not yet implemented)
