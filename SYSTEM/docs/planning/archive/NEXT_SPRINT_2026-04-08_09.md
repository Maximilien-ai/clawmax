## Priority Order

### 1. Centralized Keys / Secrets Store

Why first:
- repeated friction across partner integrations, workflows, and skills
- strongest cross-cutting UX improvement from investor feedback
- likely unlocks smoother onboarding and template apply at the same time

First slice:
- browser-local keystore page/surface
- unique key names + values
- support workspace scope vs. all-workspaces scope
- prefill matching secret requirements in:
  - template apply
  - workflow run inputs
  - partner integrations
  - skills where applicable
- allow local override without mutating the saved key
- make `Keys & Secrets` / browser vault the source of truth for provider and visible partner keys, with Workspaces Integrations reflecting and updating that vault

Deliverable:
- first working keystore plus prefill integration in at least templates + workflows

### 2. Onboarding → Template Recommendation Flow

Why second:
- direct investor signal
- strong “first value” story
- can build on the centralized key/secret groundwork

First slice:
- onboarding step that asks:
  - category
  - focus
  - free-form goal
- suggest top templates from:
  - tags/category metadata
  - simple semantic search / intent match
- allow quick apply or “show me more like this”

Deliverable:
- onboarding recommender that gets a user to 2–5 good templates quickly

### 3. Workflow Run-Level Input Overrides

Why third:
- high-value product improvement
- clear use case for restarting/unblocking pipelines without editing workflow definitions

First slice:
- trigger/restart workflow with run-scoped instructions
- store these separately from persistent workflow body
- expose them in execution records
- pass them into all participants for that run only

Deliverable:
- “Run with extra instructions” flow for manual workflow starts

### 4. Small-Business Marketing Template Pack

Why fourth:
- concrete domain demand from demo
- good showcase templates once onboarding/recommendation improves

First slice:
- one budget/planning desk template
- one cross-channel prioritization template
- one ads-performance review template

Candidate focus:
- Instagram / Facebook
- YouTube
- Google News / Google Ads

Expected requirements:
- optional browser-local API keys
- optional uploaded historical performance data

Deliverable:
- 2–3 suggested starter templates designed to be refined by the user
