# Blaxel App Studio

Use this template when you want a small team of agents to take an app idea, shape the first implementation plan, and prepare a sandbox-first deploy path through Blaxel.

Best for:
- app or service prototypes
- sandbox-first demos
- build and deploy experiments

Recommended integrations:
- Blaxel enabled in Workspaces Integrations
- GitHub if you want repo-backed build coordination
- Workspace File System for drafts and artifacts

Typical flow:
1. Kick off the app brief and sandbox targets
2. Produce a first build pass
3. Define the sandbox deploy plan
4. Run QA and ship review
