# Redis Memory Research Desk

Use this template when you want agents to do real research work while explicitly deciding what should persist in fast shared memory for later runs.

Best for:
- research programs with repeated follow-up
- analyst desks that need reusable recall
- decision briefs with persistent context

Recommended integrations:
- Redis enabled in Workspaces Integrations
- Senso if you also want evidence ingestion and retrieval
- Workspace File System for artifacts and drafts

Typical flow:
1. Define the research question and memory goal
2. Run the evidence sweep
3. Design the Redis memory plan
4. Produce the final synthesis and reuse guidance
