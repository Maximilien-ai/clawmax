Proposal template for taking a robotics idea from platform choice and system design through prototyping, integration, and demo readiness. Early idea only and intended as a customizable starter.

## Agents

| id | name | role | tags | skills |
|----|------|------|------|--------|
| robotics-lead | Robotics Lead | Robotics lead — defines the project goal, keeps the scope coherent, and synthesizes the final build direction. | lead, robotics, systems | |
| platform-architect | Platform Architect | Platform specialist — evaluates robotics hardware/software options, Qualcomm AI Hub paths, Arduino fit, and deployment constraints. | robotics, platforms, embedded | |
| prototype-builder | Prototype Builder | Build specialist — turns the chosen architecture into a working prototype plan, implementation tasks, and integration steps. | robotics, prototype, build | |
| controls-engineer | Controls Engineer | Controls specialist — thinks through sensing, control loops, feedback, runtime timing, and system reliability. | robotics, controls, systems | |
| demo-operator | Demo Operator | Demo specialist — prepares scenario scripts, validation passes, and a clean demo story for the prototype. | robotics, demo, validation | |

## Communities

- **Robotics Lab** — Shared coordination space for robotics platform decisions, system design, implementation, and demo readiness.

## Groups

- **Platforms** — Platform selection, hardware constraints, Qualcomm AI Hub fit, and Arduino considerations. (Robotics Lab)
- **Architecture** — System design, control loops, runtime architecture, and integration boundaries. (Robotics Lab)
- **Build** — Prototype implementation, code, firmware, and integration tasks. (Robotics Lab)
- **Demo** — Demo scenarios, validation, presentation notes, and readiness checks. (Robotics Lab)
- **Status** — Current scope, blockers, integration risks, and next milestones. (Robotics Lab)
