# New Team Member Onboarding

## Objective
Guide a new team member through their first week onboarding tasks.

## Instructions

Welcome to the team! This workflow will help you get set up and oriented. Please work through the following sections and report your progress.

### Day 1: Account Setup & Access
- [ ] Email and calendar access configured
- [ ] Slack/communication platform access granted
- [ ] GitHub/code repository access granted
- [ ] Development environment setup instructions received
- [ ] Security training completed
- [ ] Password manager setup (1Password, LastPass, etc.)

### Day 1-2: System Configuration
- [ ] Development environment installed and working
- [ ] Can clone and build the codebase
- [ ] Database access configured
- [ ] API keys and credentials obtained
- [ ] Development tools installed (IDE, debugger, etc.)
- [ ] Successfully ran test suite locally

### Day 2-3: Knowledge Transfer
- [ ] Read team documentation and onboarding guide
- [ ] Watched architecture overview video/presentation
- [ ] Reviewed coding standards and style guide
- [ ] Read deployment and release process documentation
- [ ] Familiarized with project management tools (Jira, Linear, etc.)

### Day 3-4: Team Integration
- [ ] Met with each team member (1-on-1 intro calls)
- [ ] Attended team standup meetings
- [ ] Joined relevant Slack channels/groups
- [ ] Added to team calendar for recurring meetings
- [ ] Understood team workflows and communication norms

### Day 4-5: First Tasks
- [ ] Assigned first onboarding ticket/task
- [ ] Completed first code review
- [ ] Submitted first pull request
- [ ] Participated in code review for teammate
- [ ] Asked clarifying questions in team channels

### End of Week 1: Reflection
**What went well?**

**What was confusing or unclear?**

**What documentation or resources would have been helpful?**

**Questions for your onboarding buddy or manager:**

## Output Format
Please update this checklist with ✅ for completed items and provide your end-of-week reflection answers.
