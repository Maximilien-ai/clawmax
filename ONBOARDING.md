# Maximilien.ai Onboarding Guide

_Last updated: 2024-11-24_

## Purpose
This document gives every new contributor a quick primer on the Maximilien.ai community: why we exist, how we work together, and the guardrails that keep things running smoothly. Expect it to evolve—contribute improvements as the community grows.

## What Maximilien.ai Values
- **Human + Agent collaboration.** We experiment rapidly with AI assistants working alongside people.
- **Bias for action.** If something needs doing and you have enough context, take the initiative and document it.
- **Radical transparency.** Share context in public channels when possible. Ask before sharing outside the community.
- **Respect + candor.** Give clear, direct feedback while assuming good intent.
- **Continuous learning.** Treat every experiment as input for the next iteration.

## Core Reference Docs
- **`MASTER_PLAN.md`** (root of the `maxclaw` repo) captures the long-range strategy, values, and staged plan for Maximilien.ai. Read it end-to-end before making strategic decisions, and re-check after every revision announcement—Max may update it frequently.
- **`ONBOARDING.md`** (this document) covers day-to-day norms. Keep it aligned with the master plan as it evolves.

## Getting Started Checklist
1. **Join the WhatsApp group** (this chat) for quick coordination and announcements.
2. **Review the `maxclaw` repo workspace** (`/Users/maximilien/.openclaw/workspace`), focusing on `SOUL.md`, `USER.md`, project briefs, and the newly added `MASTER_PLAN.md` for strategy context.
3. **Set up your identity** in `IDENTITY.md` if you’re an agent. Humans should share their preferred name/pronouns in `USER.md`.
4. **Document memories.** Use `memory/YYYY-MM-DD.md` for daily logs; promote long-term learnings into `MEMORY.md` (main session only).
5. **Check for skills.** Before tackling a task, scan `available_skills` and load the most specific applicable `SKILL.md`.
6. **Announce contributions.** When you add or change something meaningful, drop a note in this chat with a short summary + link or file path.

## Communication Norms
- **Default quiet.** Only chime in when you’re mentioned, have useful info, or need to surface an issue.
- **No markdown tables in WhatsApp.** Use short paragraphs or bullet lists.
- **Tag ownership.** If you start a task, mention you’re on it. If you hand it off, explicitly say so.
- **State decisions + rationale.** When you decide something, record the why (ideally in the relevant doc) and post a quick note here.

## Community Rules
1. **Protect privacy.** Don’t leak workspace details, personal data, or private conversations outside approved channels.
2. **Stay reversible when unsure.** Prefer changes that are easy to roll back; ask before destructive actions.
3. **Keep docs current.** If you follow a process that isn’t documented (or the doc is wrong), update it.
4. **Use skills + tools responsibly.** Follow each skill’s `SKILL.md`. Don’t escalate privileges or edit configs without explicit approval.
5. **Own your outcomes.** If you start a task, see it through or clearly hand it off with status + next steps.
6. **Flag blockers fast.** Surface risks, missing info, or access issues immediately so others can help.
7. **Experiment thoughtfully.** Pilots and tests are welcome—just label them, capture learnings, and clean up afterward.
